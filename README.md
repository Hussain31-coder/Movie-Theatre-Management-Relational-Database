# Movie Theatre Management System 🎬

A production-ready database management system implemented and optimized within **MySQL Workbench**. This project models the complete data infrastructure of a modern cinema, mapping complex operations from an abstract Entity-Relationship Diagram (ERD) into fully optimized physical tables and business-critical analytical workflows.

---

## 📌 Features

* **Show & Ticketing Management:** Tracks showtimes, ticket categories, pricing structures, and real-time seating availability.
* **Operational Asset Management:** Manages multiple theatre branches, individual viewing screens, seating capacities, and hardware specifications.
* **Customer Loyalty Framework:** Stores demographic profiles, automates age tracking, and links customer accounts to purchase histories.
* **Point of Sale Integration:** Unifies ticket purchases and concession snack sales under centralized transaction logs.
* **Staff Infrastructure Management:** Organizes employee roles, designations, shifts, salaries, and specific venue assignments.

---

## 🗺️ Database Architecture

### 1. Conceptual Design (ERD)
The database structure relies on an Entity-Relationship Diagram that captures the synergy between different cinema sectors:
* **Core Entities:** `Movie`, `Show`, `Ticket`, `Customer`, `Payment`, `Snack`, `Seat`, `Screen`, `Theatre`, and `Staff`.
* **Key Synergies:** Movies link to Shows ➔ Shows generate Tickets ➔ Customers purchase Tickets & Snacks via Payments.

### 2. Relational Schema
The conceptual design maps directly into normalized relational tables. This design eliminates data redundancy and maintains strict data integrity across transactions.

## 💻 Implementation & Queries

The physical layout and business logic are fully written, deployed, and tested inside **MySQL Workbench**. 

### Core Database Components
* **DDL Scripts:** Structured schemas defining tables, data types, primary keys, and complex foreign key constraints.
* **DML Scripts:** Mock transactional records mimicking daily theatre activities.

### Business-Critical Intelligence Reports
The repository contains an advanced analytical suite of SQL scripts engineered to answer key operational questions:

```sql
-- Example: Total Revenue Generated by Ticket Sales & Snacks Combined
SELECT 
    (SELECT SUM(amount) FROM payment WHERE transactionStatus = 'Completed') AS Total_Ticket_Revenue,
    (SELECT SUM(price * size) FROM snack WHERE isAvailable = 1) AS Potential_Concession_Revenue;
```

* **Revenue Performance:** Tracks total daily and monthly earnings from combined ticket sales and concession counters.
* **Capacity Analytics:** Evaluates occupancy rates to identify the most popular time slots, screen types, and movie genres.
* **Operational Audits:** Monitors transaction statuses, processing discount anomalies, and ticket cancellations.

## 🚀 Getting Started

### Prerequisites
* MySQL Server (v8.0+)
* MySQL Workbench

### Installation
* 1. Clone this repository:
   ```bash
   git clone
 https://github.com/Hussain31-coder/Movie-Theatre-Management-Relational-Database
    ```
* 2. Open **MySQL Workbench**.
* 3. Import and execute the initialization script located in `/database/schema.sql` to generate the tables.
* 4. Populate the database by running `/database/seed_data.sql`.
* 5. Browse and test the operational analytics inside `/queries/business_reports.sql`.
